Проверенные факты по регуляторике и бизнес-процессам. Источники с датой проверки — июнь 2026.

## 1. Честный знак — обязателен для розничной торговли чаем

- **Эксперимент:** 1 июня – 31 августа 2025, ПП РФ № 809 от 31.05.2025.
- **Обязательная маркировка чая** (чай, чайные напитки, мате, экстракты): с **1 апреля 2026** — для производителей и импортёров. На 16.06.2026 уже действует.
- **Кофе и цикорий:** с 1 июня 2026.
- **Вывод из оборота на кассе при продаже + объёмно-сортовой учёт (ОСУ):** с **1 июня 2027** (переходный ОСУ до 30.09.2028).
- **Разрешительный режим на кассе для бакалеи:** с 1 марта 2027.
- **Остатки** до старта — продаются без кодов до конца срока годности.

Обязанности розничного продавца (вкл. интернет-магазин):
1. Регистрация в Честном знаке + УКЭП.
2. Приёмка маркированного товара по ЭДО (УПД с кодами) — нужен оператор ЭДО.
3. С 01.06.2027 — вывод из оборота при продаже через ОФД/онлайн-кассу (ФФД 1.2).
4. Касса ФФД 1.2 + разрешительный режим (проверка кода на кассе) с 2027.

## 2. Честный знак пошагово — после получения коробки

**Приёмка (одинаково):** УПД с кодами по ЭДО → сверка Data Matrix с пачек → подтверждение приёмки → коды на твоём балансе ЧЗ.

**Развилка по фасовке (выбрана модель B + C):**
- **A. Продаём как пришло** — ничего сверх приёмки; вывод из оборота на кассе при продаже (с 01.06.2027).
- **B. Развес под заказ** — при фасовке вывод из оборота с причиной «Фасовка», привязка к партии, порции поштучно НЕ маркируем.
- **C. Перефасовка в свои фиксированные пачки** — вскрытие → вывод старого кода → эмиссия НОВЫХ кодов (заказ в ЧЗ, ~60 коп/код + НДС, печать Data Matrix). Ты становишься фасовщиком: отдельная роль/регистрация.

Источники приёмки: https://sbis.ru/articles/marking/acceptance , https://astral.ru/aj/elem/edo-pri-poluchenii-tovara-kak-prinyat-markirovannuyu-produktsiyu/

## 3. Бронь товара до оплаты (защита от двойной продажи)

Механизм «резерв»: при создании заказа остаток резервируется. `Свободный остаток = физический − резерв`, витрина показывает свободный. Второй покупатель не закажет зарезервированную пачку. Резерв снимается по таймауту неоплаты/отмене. Источник правды по остаткам — МойСклад (а не витрина), иначе race condition при всплеске заказов. Штучные позиции (редкий пуэр) — строго поштучно.

## 4. FIFO

First In First Out — отгружается более ранняя партия, товар не залёживается до истечения срока. Для зелёного/свежего чая обязателен; для пуэра обратное (выдержка ценна). В МойСкладе — партионный учёт + выбор партии при отгрузке.

## 5. Роли: кладовщик vs менеджер = две системы

- Реальное движение товара (поступление, фасовка, списание, инвентаризация) — **МойСклад**.
- Путь заказа (статус, клиент, оплата, рассылка, лояльность) — **RetailCRM**.
- Связка двусторонняя и автоматическая, дублировать руками не нужно. НО физически это **два кабинета**. «Одно окно» даёт только ERP-ядро (1С/МойСклад как центр) — на старте осознанно отложено. Для соло-владельца на старте терпимо, при росте — боль (развилка фазы 2).

## 6. Наборы (bundles) для рекламы

- **Сайт:** тип товара «комплект/набор» (Аспро/Битрикс умеют) — отдельная карточка из N компонентов, цена-стимул; опц. конструктор «собери набор» (кастомный модуль).
- **Склад/CRM:** МойСклад «Комплекты» (остаток по компонентам) ИЛИ набор как отдельный фасуемый SKU. Промо — правила корзины/акции/промокоды в RetailCRM.
- **ЧЗ:** при продаже набора с маркированными компонентами выводятся коды всех компонентов.
- Механики: «набор недели», конструктор, пробник в подарок от суммы.

## 7. Возвраты и брак

Пищевой товар надлежащего качества возврату НЕ подлежит; брак/недовоз/пересортица — да.
Процесс: заявка клиента → возврат/рекламация в RetailCRM → решение (деньги/замена) → возврат товара: оприходование в МойСклад или списание (если уничтожается) → возврат платежа ЮKassa + чек возврата 54-ФЗ → ЧЗ: код вернуть в оборот либо списать с причиной.
Функционал: статус «возврат» в воронке CRM, возвратный документ в МойСкладе, возврат платежа ЮKassa, чек возврата на кассе, обработка кода в ЧЗ.

## 8. Выбор склада: почему МойСклад (проверено 16.06.2026)

Выбран под требования рабочего продукта: чай с партиями, обязательная маркировка, связка с RetailCRM, один владелец на старте.

Причины:
1. Партионный учёт из коробки — «год сбора = отдельная позиция», партии с датами без доработок.
2. Бесшовная интеграция с Честным знаком — приёмка/выбытие/фасовка кодов прямо из МойСклада, без захода в ЛК ЧЗ.
3. Нативная связка с RetailCRM — обмен заказами/остатками настроенный, не самопис.
4. Облако, быстрый старт — без сервера и штатного админа.
5. Цена входа ниже 1С.

Альтернативы:
- **1С (УТ/Розница)** — ближайший аналог, мощнее и без потолка по росту (производство, сети, опт), но дороже на входе (лицензии + ИТС + интегратор), нужен админ. Целевая система при масштабировании фасовки в цех / развитии опта.
- **Контур.Маркет** — силён в маркировке/ЭДО (один вендор), но слабее как ядро интернет-магазина с партиями/API.
- **Склад внутри RetailCRM** — один кабинет, но складская часть слабее для партий/фасовки чая.
- **SUBTOTAL/ЕКАМ/Класс365** — нишевые, меньше экосистема.

Отзывы (реальные): минусы — поддержка только в чате и медленная; автопополнение по продажам руками, дубли кодов позиций при импорте; жалобы на расхождения по безналу/предоплатам; цена «слегка завышена»; зависимость от интернета. Плюсы — богатый функционал, гибкие настройки, лидер сегмента облачного учёта для онлайн-розницы.

Best practices: остатки/резерв держит МойСклад (источник правды), сайт отражает; партии с датами обязательны; маркировку вести из МойСклада; тарифы по реальной команде; данные держать выгружаемыми на случай перехода на 1С при росте опта/производства; GetMark/Контур как прослойка маркировки для сложной фасовки (ветка C).

## 9. Что упущено в исходном предложении (12 пунктов)

По приоритету (★ — критично, может сделать запуск нелегальным/убыточным):
1. ★ **Маркировка Честный знак** — нет вообще: регистрация, ЭДО, касса ФФД 1.2, выбытие при фасовке, эмиссия кодов (ветка C).
2. ★ **Закупка/импорт** — откуда чай (прямой импорт из Китая / параллельный импорт / опт). Определяет себестоимость и юр. оформление.
3. ★ **Сертификация** — декларации ТР ТС, фитосанитария.
4. **Фасовка как процесс** — развес + свои пачки, весы, печать этикеток, привязка к партиям/ЧЗ.
5. **Сроки годности и FIFO** — учёт по партиям с датами, уценка/списание.
6. **Дегустационные наборы/пробники** — драйвер первой покупки, отдельный тип товара.
7. **Контент-машина** — статьи/заваривание/история сорта = SEO-трафик.
8. **Возвраты и брак** — процесс + чек возврата (54-ФЗ).
9. **Юрлицо и налоги** — ИП/ООО, режим налогообложения.
10. **Опт / B2B** — другой прайс, мин. заказ.
11. **Многоканальность** — Ozon/WB/Я.Маркет, Telegram, офлайн.
12. ★ **Юнит-экономика** — средний чек, маржа, точка безубыточности под бюджет 1.2 млн + 8–11к/мес.

# Интернет-магазин китайского чая

Рабочий интернет-магазин китайского чая (150–200 SKU, квиз-сегментация, полный контур e-commerce). Здесь — техническое предложение, рисёч регуляторики и архитектура бизнес-процессов.

## Стек: предложение vs рекомендация

**В исходном предложении** (`proposal.html`): Битрикс+Аспро Next · RetailCRM · МойСклад, под ключ ~1.2 млн ₽.

**Рекомендация после дип-рисёча** (`stack-research.md`), под 80–100 SKU, настоящий кастомный дизайн и вайбкодинг фронта самим владельцем:
- **Витрина:** свой фронт (Next / Astro / SvelteKit), вайбкод — полная свобода дизайна. Средний путь без склейки с нуля — CS-Cart (полный код на своём сервере). InSales отвергнут: витрины шаблонные, кастом упирается в потолок Liquid.
- **Заказ/клиенты:** RetailCRM (заказы по API, сегменты, квиз→сегмент, лояльность)
- **Склад:** МойСклад + опция «Маркировка» (партии, сроки/FEFO, коды ЧЗ)
- **Платежи:** ЮKassa (карты+СБП), фискализация 54-ФЗ
- **Маркировка:** Честный знак, обязательна для чая с 01.04.2026; фасовка A (из коробки) + B (развес), своих пачек нет. Compliance — в бэк-офисе (МойСклад/касса), не в витрине
- **Доставка:** СДЭК / Boxberry / Почта / Яндекс.Доставка

## Статус

Стадия проектирования рабочего продукта: предложение, рисёч и архитектура готовы; стек выбран (свой фронт + RetailCRM + МойСклад + ЮKassa). Следующий шаг — реализация.
